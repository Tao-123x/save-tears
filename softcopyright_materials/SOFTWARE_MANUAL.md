## 2. 开发背景

在宿舍、公寓、校园生活区等场景中，用水数据、水费账单和住户身份信息通常分散管理，容易出现统计不及时、房间关联不清晰、用户查询不便等问题。为解决上述问题，开发了 Save Tears 节水管理系统。该系统以房间号为核心索引，将用户身份、用水信息、污水浊度数据及水费账单进行统一关联，并通过微信小程序提供用户访问入口，通过 Python 后端提供业务接口服务，从而形成一套可部署、可扩展的节水信息管理系统。

## 3. 开发目的

本软件的开发目的主要包括：

1. 建立统一的房间级用水信息管理机制。
2. 为住户提供便捷的登录、查询和数据查看入口。
3. 为管理方提供结构化的用户、用水、污水和账单数据基础。
4. 为后续节水分析、运营统计和管理扩展提供底层系统支撑。

## 4. 软件总体概述

本软件采用前后端分离架构，由微信小程序客户端、后端接口服务和数据库存储三部分组成。小程序负责用户交互和数据展示，后端负责业务逻辑处理和数据读写，数据库负责保存用户、用水、污水浊度及账单信息。

系统当前已实现以下核心业务闭环：

- 用户注册
- 用户登录
- 房间号绑定
- 用水数据提交与查询
- 污水浊度数据提交与查询
- 水费账单提交与查询

## 5. 运行环境

### 5.1 开发环境

- 操作系统：macOS / Windows / Linux

- 后端开发语言：Python

## 6. 系统架构

### 6.1 架构说明

系统采用三层结构：

1. 表现层
   - 微信小程序页面，负责用户输入、页面跳转、信息展示和本地缓存。
2. 服务层
   - FastAPI 接口服务，负责注册登录、业务数据读写和错误处理。
3. 数据层
   - SQLAlchemy ORM + SQLite/MySQL，负责数据持久化。

### 6.2 架构特点

1. 通过房间号统一关联用户与业务数据。
2. 前后端通过标准接口通信，便于单独维护和扩展。
3. 默认使用 SQLite 便于本地部署，同时支持迁移到 MySQL。

## 7. 功能模块说明

## 7.1 用户认证模块

用户认证模块实现用户注册与登录功能。

主要功能：

- 注册新用户
- 校验用户名是否重复
- 登录身份验证
- 缓存用户基本信息
- 支持记住用户名

功能流程：

1. 用户在注册页填写用户名、密码和房间号。
2. 前端提交注册请求到后端。
3. 后端检查用户名唯一性，写入用户数据。
4. 用户注册成功后跳转登录页。
5. 用户在登录页输入账号密码。
6. 后端完成登录校验并返回用户名、房间号、角色信息。
7. 小程序在本地缓存用户信息，供业务页面读取。

## 7.2 房间绑定模块

房间绑定模块通过房间号将用户与业务数据关联。

主要功能：

- 注册时填写房间号
- 登录后缓存房间号
- 数据查询页面自动读取房间号
- 用房间号作为用水、污水、账单数据的检索条件

该模块使系统具备宿舍、公寓等房间化场景的适配能力。

## 7.3 用水数据管理模块

用水数据管理模块负责保存和展示用户所属房间的用水信息。

主要功能：

- 提交用水数据
- 查询指定房间的用水记录
- 在小程序中展示数据编号、流量值和时间戳

业务流程：

1. 用户登录系统。
2. 页面读取本地缓存中的房间号。
3. 前端调用用水查询接口。
4. 后端按房间号返回用水记录集合。
5. 页面列表展示结果。

## 7.4 污水浊度管理模块

污水浊度管理模块用于保存和展示房间对应的污水浊度信息。

主要功能：

- 提交污水浊度数据
- 查询指定房间的污水浊度记录
- 展示浊度值和采集时间

该模块为后续水质监测和异常识别扩展提供基础数据结构。

## 7.5 水费账单管理模块

水费账单管理模块用于保存和查询账单信息。

主要功能：

- 提交账单数据
- 按房间号查询账单
- 展示账单金额和账单月份

该模块可用于住户自助查询和后续账单统计扩展。

## 7.6 首页展示模块

首页展示模块提供节水主题的欢迎界面和可扩展入口。

当前实现内容包括：

- 用户欢迎信息展示
- 节水影响卡片
- 生态任务展示
- 快捷入口区域

该模块当前以展示层为主，适合在说明书中描述为用户首页展示模块，不建议写为完整算法推荐系统。

## 7.7 个人中心模块

个人中心模块用于展示用户信息和功能入口。

当前实现内容包括：

- 读取本地缓存中的用户名
- 展示用户头像、用户名和功能卡片
- 提供图表、设备、个性化等扩展入口占位

## 7.8 基础管理界面模块

系统包含基础管理中心界面，当前实现管理端页面结构和操作入口。

当前实现内容包括：

- 顶部导航切换
- 搜索框
- 统计卡片
- 列表展示区域
- 操作菜单弹窗

说明：

该模块当前更适合描述为基础管理界面或管理端初版页面，不建议在申报时表述为完整审批流系统。

## 8. 数据库设计

## 8.1 用户表 `users`

字段说明：

- `id`：用户主键
- `username`：用户名
- `password_hash`：密码字段
- `room_number`：房间号
- `role`：角色标识

## 8.2 用水数据表 `water_flow`

字段说明：

- `id`：主键
- `room_number`：房间号
- `flow_rate`：流量值
- `timestamp`：采集时间

## 8.3 污水浊度表 `sewage_turbidity`

字段说明：

- `id`：主键
- `room_number`：房间号
- `turbidity_value`：浊度值
- `timestamp`：采集时间

## 8.4 水费账单表 `water_bill`

字段说明：

- `id`：主键
- `room_number`：房间号
- `amount`：账单金额
- `month`：账单月份

## 9. 技术特点

本软件的主要技术特点如下：

1. 基于微信小程序提供用户端访问入口，便于在移动端使用。
2. 采用 `FastAPI + SQLAlchemy` 构建接口服务和数据模型，结构清晰，易于维护。
3. 采用房间号作为业务主键之一，使用户与数据形成明确绑定关系。
4. 采用统一 API 封装方式，便于扩展新的业务接口。
5. 支持 `SQLite` 和 `MySQL` 两类数据库，便于从开发环境迁移至正式环境。

## 10. 操作流程说明

### 10.1 用户注册

1. 打开小程序注册页。
2. 输入用户名、邮箱、密码、确认密码和房间号。
3. 勾选同意条款。
4. 点击注册按钮。
5. 系统校验输入合法性并调用注册接口。
6. 注册成功后跳转登录页。

### 10.2 用户登录

1. 打开登录页。
2. 输入用户名和密码。
3. 点击登录按钮。
4. 系统调用登录接口。
5. 登录成功后缓存用户信息并跳转首页。

### 10.3 查看用水数据

1. 用户登录系统。
2. 进入“我的用水表”页面。
3. 页面自动读取房间号。
4. 用户点击刷新数据。
5. 页面展示该房间的历史用水数据。

### 10.4 查看污水浊度数据

1. 用户登录系统。
2. 进入“我的污水表”页面。
3. 页面读取房间号并调用查询接口。
4. 页面展示污水浊度记录。

### 10.5 查看水费账单

1. 用户登录系统。
2. 进入“我的水费账单”页面。
3. 页面读取房间号并调用账单查询接口。
4. 页面展示账单记录。
