## 2. 软件类型与用途

本软件是一个面向宿舍、公寓或园区场景的节水管理系统，采用微信小程序前端与 Python 后端接口服务相结合的方式，实现住户信息管理、房间维度的数据绑定、用水数据查询、污水浊度查询、水费账单查询等功能。软件主要服务于节水管理、用水透明化、房间级数据追踪与基础运营管理。

## 4. 软件功能说明

### 4.1 用户注册与登录

系统支持新用户注册与已有用户登录。注册时，用户填写用户名、密码和房间号，后端完成用户名唯一性校验并写入用户数据；登录成功后，小程序本地缓存用户身份信息，用于后续房间号绑定和业务页面访问。

对应实现：

- 后端注册接口：`/register`
- 后端登录接口：`/login`
- 小程序页面：
  - `save_tears_miniprogram/src/pages/register/index.vue`
  - `save_tears_miniprogram/src/pages/login/index.vue`

### 4.2 房间号绑定与住户身份关联

系统通过房间号将用户身份与业务数据关联。用户登录后，小程序从本地缓存读取用户信息，并自动携带用户所属房间号，用于查询该房间的用水、污水和账单数据。

这一设计使系统具备“按房间管理”的业务特点，适合宿舍、公寓、楼栋等场景。

### 4.3 用水数据管理

系统支持提交和查询用水数据。后端提供水流量数据写入接口与按房间号查询接口，小程序端提供“我的用水表”页面，展示该房间历史用水记录，包括数据编号、流量值和时间戳。

对应实现：

- 后端接口：
  - `POST /water_flow`
  - `GET /water_flow/{room_number}`
- 小程序页面：
  - `save_tears_miniprogram/src/pages/water-flow/index.vue`

### 4.4 污水浊度数据管理

系统支持污水浊度数据的提交与查询。后端保存污水浊度值和采集时间，小程序端以房间号为索引展示历史浊度记录，便于了解房间对应区域的污水状态。

对应实现：

- 后端接口：
  - `POST /sewage_turbidity`
  - `GET /sewage_turbidity/{room_number}`
- 小程序页面：
  - `save_tears_miniprogram/src/pages/sewage-turbidity/index.vue`

### 4.5 水费账单管理

系统支持水费账单数据提交与查询。后端保存房间号、账单金额和账单月份，小程序端提供“我的水费账单”页面，用户可以查看账单记录。

对应实现：

- 后端接口：
  - `POST /water_bill`
  - `GET /water_bill/{room_number}`
- 小程序页面：
  - `save_tears_miniprogram/src/pages/water-bill/index.vue`

### 4.6 用户首页与个人中心展示

系统提供用户首页和个人中心页面，用于展示欢迎信息、节水主题卡片、生态任务、个人入口、图表占位区域等内容。当前这部分偏向用户端展示层和后续扩展入口，已有基本界面和本地用户信息读取逻辑。

对应页面：

- `save_tears_miniprogram/src/pages/home/index.vue`
- `save_tears_miniprogram/src/pages/profile/index.vue`

### 4.7 管理界面基础展示

系统包含基础管理中心页面，当前已实现搜索框、统计卡片、申请人列表和操作菜单等界面结构，可视为后台管理功能的原型基础或初版界面。由于当前代码未与真实审批数据完全联动，申报时不建议写成完整审批业务系统。

对应页面：

## 5. 软件架构说明

本软件采用前后端分离架构，整体分为小程序客户端、后端接口服务层、数据持久化层三部分。

### 5.1 小程序客户端

小程序客户端基于 `uni-app` 框架开发，使用 `Vue 3` 作为视图层框架，负责用户登录注册、页面展示、数据请求、结果渲染与本地缓存。

### 5.2 后端接口服务层

后端基于 `FastAPI` 开发，负责提供统一 HTTP 接口，完成用户注册登录、用水数据录入、污水浊度录入、水费账单录入及按房间号查询等业务逻辑。

### 5.3 数据持久化层

数据持久化通过 `SQLAlchemy` ORM 实现，默认使用 `SQLite` 数据库，亦支持通过环境变量切换到 `MySQL`。后端在启动时自动完成基础数据表创建。

### 6.2 用水数据表 `water_flow`

### 6.3 污水浊度表 `sewage_turbidity`

主要字段：

- `id`：主键
- `room_number`：房间号
- `turbidity_value`：浊度值

### 6.4 水费账单表 `water_bill`

主要字段：

- `room_number`：房间号

- `month`：月份

## 7. 技术特点

结合现有代码，本软件可归纳出以下技术特点：

1. 采用前后端分离架构，小程序与后端通过标准 HTTP 接口通信，便于后续独立升级。
2. 以房间号为主索引组织业务数据，使用户、账单和监测数据形成统一关联。
3. 使用 ORM 模型管理数据库结构，降低数据库适配复杂度。
4. 默认支持本地 SQLite 快速部署，同时保留 MySQL 扩展能力，方便从开发环境迁移到生产环境。
5. 小程序端封装统一 API 调用层，便于后续增加接口和切换后端地址。
